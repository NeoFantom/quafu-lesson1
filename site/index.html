01 · Quafu first public lesson

从网页登录到第一个云端量子任务：一节课跑通 Quafu。

这份教程面向第一次使用 Quafu 的学员。目标是建立一条可复现的最小闭环：看懂平台、准备 token、写电路、选择后端、提交任务、读取 counts。

02 · workspace map

先把 Quafu 看成四个普通工作区。

登录后最常用的入口可以压缩成四类：Home 看资源与后端状态，Composer 画电路和生成 OpenQASM，Tasks 查任务列表与状态，User/Jupyter 管理个人信息和 JupyterLab 服务。

Home

显示个人可用 QPU、token 到期时间、任务统计，以及每台 QPU 的在线状态、队列长度、qubits、couplers、1Q/2Q error。

Composer

左侧是门操作，中央是 qubit wires，右侧/下方展示 OpenQASM 与输出态，用来建立电路直觉。

Tasks

表格字段包括 Task ID、Task Name、Chip、Status、Submit Time。第一课只要求会按 task id 找到任务。

User / Jupyter

User 页含个人信息、JupyterLab server 和密码更新；Jupyter 页可能需要等待服务启动。

证据：`sources/raw/platform/nav/extract-7.md`, `extract-11.md`, `extract-13.md`, `extract-15.md`。

04 · token is the key

token 是个人凭证：课堂只写占位符。

官方文档中的最短入口是 `from quark import Task`，再用 token 初始化任务管理器。token 用来证明你有权限向 Quafu 后端提交与查询任务。

from quark import Task

tmgr = Task("YOUR_QUAFU_TOKEN")
classroom rule

不要把真实 token 发到群里，不要写进公开代码，不要提交到仓库。教程、作业、截图里都使用 YOUR_QUAFU_TOKEN 这类占位符。

官方文档说明 token 有有效期；Home 页面也会显示 API token expiration。过期时重新刷新/申请即可。

30

官方文档写明 token 有效期为 30 天。

1000

官方文档写明每个用户每日可提交 1000 个任务，超出会排队到之后处理。

来源：Quafu ReadTheDocs Home → Quick Guide。

05 · build the first circuit

Composer 帮你把“门”和 “OpenQASM” 对上。

Composer 页面暴露了 H、X、Y、Z、S、T、RX、RY、RZ、U 等操作；默认电路有 `q[0]` 到 `q[3]` 四条线，并展示生成的 OpenQASM。

选门

先用 H 和 CX/CNOT 讲清楚叠加与纠缠，不急着铺满所有门。

放到 qubit wire

把门拖到目标 qubit；多比特门要确认控制/目标线连接正确。

读 OpenQASM

让学员看到图形电路如何变成可提交的文本电路。

看输出态

输出态区域适合用来讨论“模拟器里可直接看 state，真实后端最终看测量 counts”。

first demo

课堂演示建议用 Bell state：`h q[0]; cx q[0],q[1]; measure ...`。它短、直观，结果分布也容易解释。

06 · QuarkStudio path

Python SDK 路线：status → run → result。

ReadTheDocs 当前 Home 页的核心是 QuarkStudio。第一课只需要掌握三个方法：tmgr.status() 看后端，tmgr.run(task) 提交，tmgr.result(tid) 取回结果。

# 1. install
pip install quarkstudio  # Python >= 3.12

# 2. initialize
from quark import Task
tmgr = Task("YOUR_QUAFU_TOKEN")

# 3. inspect backends
print(tmgr.status())
circuit = """
OPENQASM 2.0;
include "qelib1.inc";
qreg q[2];
creg c[2];
h q[0];
cx q[0],q[1];
measure q[0] -> c[0];
measure q[1] -> c[1];
"""

task = {
    "chip": "Dongling",
    "name": "Lesson1Bell",
    "circuit": circuit,
    "shots": 1024,
    "options": {"compiler": "quarkcircuit", "correct": False, "open_dd": None, "target_qubits": []}
}

tid = tmgr.run(task)
print("task id:", tid)
07 · read the result

结果是一组可解释的 counts。

Quafu 返回的结果字典里，第一课最应该读的是 status 与 count。如果任务还没完成，先保存 task id，稍后再查。

字段课堂解释怎么讲
status任务当前状态先判断是否 Finished，再看数据。
count每个 bitstring 的采样次数总和约等于 shots；Bell state 重点看 00 和 11。
transpiled实际下发前的转译电路说明云端会适配具体芯片拓扑。
error错误信息失败时先看这里，再看电路索引、shots、后端状态。
qlisp调试用电路描述进阶内容，第一课只让学生知道它存在。
res = tmgr.result(tid)
print(res["status"])
print(res["count"])
08 · lesson script

推荐 70 分钟节奏：先网页，后 SDK。

  1. 0–10 min：解释 Quafu 是云端量子计算工作台，展示 Home 上的 QPU、queue、token、任务统计。
  2. 10–25 min：打开 Composer，画 Bell state，读出 OpenQASM。
  3. 25–40 min：安装 QuarkStudio，用占位符解释 token，运行 tmgr.status()。
  4. 40–55 min：提交小任务，保存 task id；强调异步执行。
  5. 55–65 min：打开 Tasks 或用 SDK 查询结果，读 status 和 count。
  6. 65–70 min：常见问题：token 过期、后端 Offline/Maintenance、队列较长、OpenQASM 索引越界。
